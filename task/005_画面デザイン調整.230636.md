# 画面デザイン調整 設計書

**LLMモデル名:** Claude Opus 4.5 (claude-opus-4-5-20251101)

---

## 設計方針

「シンプルな中に必要なものが収まり、使い手への気心がしれたもの」

過度な装飾は避け、以下の原則に基づいて改善：
- **一貫性**: ページ間でデザインを統一
- **明確さ**: 現在地と操作を明確に
- **余白**: 適切な余白で視認性向上

---

## 改善内容

### 1. ヘッダーの統一

**現状の課題:**
- トップページは青いヘッダー、他ページは白いヘッダーで統一感がない
- ナビゲーションが右端に寄っていて分かりにくい

**改善:**
- 全ページで青いヘッダーに統一
- ナビゲーションを明確にし、現在地を示す

```html
<header class="bg-blue-600 text-white shadow-md">
  <div class="max-w-7xl mx-auto px-4 py-3 flex justify-between items-center">
    <h1 class="text-xl font-bold">商品一覧</h1>
    <nav class="flex items-center space-x-4">
      <a href="/pages/products.html" class="hover:text-blue-200 font-medium">商品</a>
      <a href="/pages/categories.html" class="hover:text-blue-200">部門</a>
      <span class="text-blue-300">|</span>
      <button id="logout-button" class="hover:text-blue-200">ログアウト</button>
    </nav>
  </div>
</header>
```

### 2. ボタンの視覚的改善

**現状の課題:**
- 「再読み込み」と「反映」ボタンの役割が視覚的に区別しにくい

**改善:**
- アイコンを追加して直感的に
- 「反映」ボタンを緑色にして重要なアクションを明示

### 3. テーブルの余白調整

**改善:**
- チェックボックス列の幅を最小限に
- 行の高さを少し小さく（コンパクトに）

### 4. 検索バーにアイコン追加

**改善:**
- 検索アイコンを追加して分かりやすく
